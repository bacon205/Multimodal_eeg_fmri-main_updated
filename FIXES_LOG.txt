FIX 3.1 — Cell 2 (BridgeConfig): PW path mismatch
  PROBLEM: eeg_path_pw pointed to `.../feat/New` but data is in `.../feat`
  FIX:     Changed to `.../feat` (no /New)
  STATUS:  APPLIED (re-applied 2026-02-25, previous save was lost)

FIX 3.2 — Cell 8 (load_eeg_pw_features): Cannot cast struct to float32
  PROBLEM: `np.array(mat[k], dtype=np.float32)` fails because the .mat file
           contains a MATLAB struct with fields (Pw, f, t, band, typem_i, subj,
           A_timecourse_all), not a raw numeric array.
  ERROR:   "Cannot cast array data from dtype([('Pw', 'O'), ('f', 'O'), ...])
           to dtype('float32') according to the rule 'unsafe'"
  FIX:     - Use loadmat(f, squeeze_me=True, struct_as_record=False)
           - Extract pw_struct.Pw field specifically
           - Handle 3D (C,F,T) -> reshape to (C*F, T)
  STATUS:  APPLIED (re-applied 2026-02-25)

FIX 3.3 — Cell 8 (load_eeg_conn_features): Same struct casting issue
  PROBLEM: Same as 3.2 but for connectivity .mat files containing
           struct with fields (PLV, COH, WPLI, band, cond, subj)
  ERROR:   "Cannot cast array data from dtype([('PLV', 'O'), ('COH', 'O'),
           ('WPLI', 'O'), ...]) to dtype('float32')"
  FIX:     - Use loadmat(f, squeeze_me=True, struct_as_record=False)
           - Extract PLV, COH, WPLI matrices from struct
           - Vectorize upper triangle of each (triu_indices, k=1)
           - Concatenate into single feature vector
  STATUS:  APPLIED (re-applied 2026-02-25)

FIX 3.4 — Cell 14 (line 73): IndexError on bridge_raw_dataset[0]
  PROBLEM: When PW and CONN fail to load (fixes 3.2/3.3), no subjects have
           complete EEG data, so BridgeRawDataset.samples is empty.
           Then bridge_raw_dataset[0] raises IndexError.
  FIX:     This is a DOWNSTREAM SYMPTOM of fixes 3.2/3.3. Once PW and CONN
           load correctly, subjects will align and samples will be populated.
  STATUS:  RESOLVED by fixes 3.2 + 3.3
